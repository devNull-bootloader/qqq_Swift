☁️ Supabase-Integration — Diese Anleitung zeigt, wie du MathSpaces-Lizenzschlüssel in einer echten Datenbank verwaltest, sodass Lehrer Schüler in Echtzeit sperren können und die Keys nicht mehr nur lokal gespeichert werden.

Was du brauchst

Supabase ist kostenlos für kleine Projekte (bis 500MB, unbegrenzte Auth-User). Kein Server-Setup nötig.
1

Supabase-Projekt anlegen

Gehe zu supabase.com, erstelle ein kostenloses Konto und ein neues Projekt. Notiere dir die Project URL und den anon public key aus Settings → API.

2

Datenbank-Tabellen erstellen

Führe dieses SQL im Supabase SQL-Editor aus (SQL Editor → New query):

-- Lizenz-Tabelle CREATE TABLE licenses ( id uuid DEFAULT gen_random_uuid() PRIMARY KEY, key text NOT NULL UNIQUE, type text NOT NULL CHECK (type IN ('pro', 'teacher', 'student')), class_id text, -- NULL for pro keys; class code for teacher/student note text, -- e.g. student name or teacher name revoked boolean DEFAULT false, locked boolean DEFAULT false, -- teacher can lock a student restrictions jsonb DEFAULT '{}', -- feature restrictions for student created_at timestamptz DEFAULT now() ); -- Allow public read (the app validates keys) ALTER TABLE licenses ENABLE ROW LEVEL SECURITY; CREATE POLICY "public read" ON licenses FOR SELECT USING (true); -- Service role can write (your admin page uses the service key) CREATE POLICY "service write" ON licenses FOR ALL USING (true);
3

Supabase-Client in index.html einbinden

Füge in index.html im <head> ein:

<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2"></script>

Dann ersetze const LICENSE_SECRET und die validateLicense-Funktion durch:

// Supabase config (replace with your values) const SUPA_URL = 'https://xxxx.supabase.co'; const SUPA_KEY = 'your-anon-public-key'; const supabase = window.supabase.createClient(SUPA_URL, SUPA_KEY); async function validateLicenseOnline(key) { const { data, error } = await supabase .from('licenses') .select('*') .eq('key', key.trim().toUpperCase()) .single(); if (error || !data) return null; if (data.revoked || data.locked) return null; return { type: data.type, classId: data.class_id, restrictions: data.restrictions || {}, note: data.note }; }
4

License Manager mit Supabase verbinden

Am Anfang von license-manager.html die Konstanten setzen:

const SUPA_URL = 'https://xxxx.supabase.co'; const SUPA_SERVICE_KEY = 'your-service-role-key'; // GEHEIM – nicht im Frontend!

⚠️ KRITISCH: Den Service Role Key darf niemals im Client-seitigen Code stehen. Er gibt vollen Datenbankzugriff ohne Row-Level-Security. Hoste den License Manager als geschützte Server-seitige Funktion (z.B. Vercel/Netlify Serverless Function hinter Authentication) – nicht als statische HTML-Datei. Alternativ: Keys nur über den Supabase-Dashboard manuell eintragen.

5

Lehrer-Echtzeit-Kontrolle

Mit Supabase Realtime kann der Lehrer einen Schüler in Echtzeit sperren:

// In der App: auf Änderungen am eigenen Key hören supabase .channel('license-watch') .on('postgres_changes', { event: 'UPDATE', schema: 'public', table: 'licenses', filter: `key=eq.${currentKey}` }, payload => { if (payload.new.locked || payload.new.revoked) { deactivateLicense(); // lock the user out immediately } applyRestrictions(payload.new.restrictions); }) .subscribe();
6

Schüler-Klassen-Kanal abonnieren

Damit Klassen-weite Einschränkungen sofort wirken, abonniere auch Änderungen an der Lehrer-Zeile:

supabase .channel('class-watch') .on('postgres_changes', { event: 'UPDATE', schema: 'public', table: 'licenses', filter: `class_id=eq.${myClassId}&type=eq.teacher` }, payload => { applyRestrictions(payload.new.restrictions); }) .subscribe();
